Desktop App

Install on your PC

A native Windows installer with a custom icon and desktop shortcut. Download once, use forever.

1

Install Node.js (one-time prerequisite)

AI Stack requires Node.js to run OpenCode and OpenChamber. If you don't have it yet:

Download and install from nodejs.org — choose the LTS version. Restart your PC after installing.

To verify it's installed, open Command Prompt or PowerShell and run:

node --version

✓ You should see something like v20.x.x. Anything v18 or higher works.

2

Install OpenCode & OpenChamber globally

AI Stack launches OpenCode and OpenChamber from your system. Install both with one command — open Command Prompt or PowerShell and run:

npm install -g opencode-ai @openchamber/web

✓ This only needs to be done once. To update them later, run the same command again.

If you skip this step, AI Stack will detect the missing tools when it opens and show you a popup with the install command — you can copy it directly from there.
3

Download the AI Stack Installer (.EXE)

Click the button below to download the Windows installer directly from Google Drive:

Download AI Stack Setup.exe
Windows SmartScreen notice: Because this installer is not yet code-signed, Windows may show a blue "Windows protected your PC" popup. Click "More info" → then "Run anyway". The file is safe.
3

Run the Installer

Double-click the downloaded AI Stack Setup.exe file. The installer will:

  • Ask you to choose an installation folder (or use the default)
  • Create a desktop shortcut icon using the OpenChamber logo
  • Add AI Stack to your Start Menu
  • Complete in under 30 seconds
4

Launch and Use

Double-click the AI Stack icon on your desktop. Within a few seconds:

  • A desktop window opens with the full OpenChamber interface
  • OpenCode starts silently in the background on port 4095
  • OpenChamber connects to it and loads in the window on port 3000
  • You're ready to create a session and start coding with AI
First launch may take 10–20 seconds while OpenCode initialises its database. Subsequent launches are instant.

To quit, simply close the window. Both services shut down cleanly automatically.

5

Add your AI Provider API Key

OpenCode supports multiple AI providers. You need to add at least one API key inside OpenChamber to start chatting:

  1. Open AI Stack — the OpenChamber window loads
  2. Click Settings (gear icon, bottom left)
  3. Go to Providers tab
  4. Choose your provider (OpenAI, Anthropic, Google, etc.) and paste your API key
  5. Save — you're now ready to start a session

VPS / Web App

Deploy on a VPS

Run the full stack on any cloud server. Access OpenChamber from your laptop or phone browser — anywhere, anytime.

1

Get a VPS

You need a Linux VPS with at least 1 GB RAM. Any of these providers work perfectly:

DigitalOcean
Hetzner (cheapest)
Vultr
Linode
AWS EC2
Google Cloud

Ubuntu 22.04 or 24.04 recommended. SSH into your server before continuing.

2

Install Docker & Docker Compose

Run this on your VPS to install Docker in one command:

curl -fsSL https://get.docker.com | sh

Then add your user to the docker group so you don't need sudo every time:

sudo usermod -aG docker $USER
newgrp docker
3

Download the docker-compose.yml

Download the ready-made Docker Compose configuration directly from the GitHub repository:

curl -fsSL https://raw.githubusercontent.com/Akakaui/opencode-desktop-app/main/docker/docker-compose.yml -o docker-compose.yml

✓ This file sets up both OpenCode and OpenChamber containers plus an auto-updater (Watchtower).

4

Start the Stack

Launch both services in the background with a single command:

docker compose up -d

Wait about 30–60 seconds for the containers to pull and start. Then check they're running:

docker compose ps

✓ Both ai-stack and ai-stack-watchtower should show as Up.

5

Open in Your Browser

On any device — your laptop, tablet, or phone — open a browser and go to:

http://<your-vps-ip>:3000

Replace <your-vps-ip> with your actual server IP address (find it on your VPS provider dashboard).

To access from the internet (not just LAN), make sure port 3000 is open in your VPS firewall. On Ubuntu: sudo ufw allow 3000
6

Add a Domain & HTTPS (Optional)

To access your stack at a clean URL like https://ai.yourdomain.com, add Nginx as a reverse proxy with a free SSL certificate from Let's Encrypt:

sudo apt install nginx certbot python3-certbot-nginx -y

# Configure Nginx to proxy port 3000
sudo nano /etc/nginx/sites-available/ai-stack

Add this Nginx config (replace ai.yourdomain.com with your domain):

server {
    server_name ai.yourdomain.com;
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
    }
}
sudo ln -s /etc/nginx/sites-available/ai-stack /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
sudo certbot --nginx -d ai.yourdomain.com

FAQ

Common Questions

What is the difference between this app and the ai-stack-setup.sh script?

The ai-stack-setup.sh script in the repository installs the entire multi-service developer workspace (including Open Design, Agent-Browser, tmux session structures, Nginx reverse-proxies, and SSL certificates) on a Linux VPS or Codespace. On the other hand, the desktop installer and Docker-compose configurations in this project are a lightweight launcher designed specifically to bundle and run OpenCode and OpenChamber together as a single unified tool (without design or browser-automation tools).

Does it work on macOS or Linux?

The current .exe installer is Windows only. On macOS or Linux you can run it from source by cloning the GitHub repo, running cd desktop && npm install && npm start. The VPS Docker deployment works on any OS that supports Docker.

Will it break when OpenCode or OpenChamber update?

No. AI Stack is a thin wrapper — it simply starts opencode serve and openchamber serve as background processes. It has no knowledge of their internal code. Any bug fixes, new features, or AI model additions from the OpenCode/OpenChamber teams are picked up automatically when the underlying packages update.

How do I set a password to protect my instance?

Password protection is managed natively inside OpenChamber's Settings. Open the app, go to Settings → Security, and configure a password there. AI Stack does not touch or control this — it works exactly the same as running OpenChamber standalone.

What if port 3000 or 4095 is already in use?

AI Stack checks if each port is already occupied before trying to start. If a port is in use it will skip starting that service (assuming it's already running). To use different ports, you'd need to modify the source and rebuild, or set PORT_OPENCODE / PORT_OPENCHAMBER environment variables in the Docker Compose file.

Is my AI session data stored locally?

Yes. On the desktop app, all session data is stored in your user application data folder (%APPDATA%/ai-stack-desktop on Windows). On VPS, data is stored in the ai-stack-data Docker volume on your server. Nothing is sent to any external service except your AI provider API calls.

Can multiple people use the VPS deployment?

Yes. Once deployed, anyone who has the URL and (optionally) the password you set in OpenChamber can access and use the interface from their browser. It's a shared instance — sessions are visible to all users with access unless OpenChamber has per-user isolation enabled in its settings.
